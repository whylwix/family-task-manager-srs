# Диаграммы поведения системы

## Use Case Диаграмма

### Акторы:
- **Семейный администратор** - создает семью, управляет участниками
- **Член семьи** - работает с задачами, выполняет поручения
- **Система** - автоматические действия (уведомления, напоминания)

### Основные сценарии:
1. **Управление семьей**: Приглашение, удаление участников
2. **Управление задачами**: Создание, редактирование, делегирование
3. **Выполнение задач**: Отметка выполнения с подтверждением
4. **Просмотр и фильтрация**: Фильтры по статусу, категории, приоритету
5. **Система вознаграждений**: Начисление баллов за выполненные задачи

## Диаграммы последовательности

### 1. Создание семейной задачи
Процесс создания задачи в рамках семьи с уведомлением назначенного

### 2. Отметка задачи выполненной
Сценарий завершения задачи с проверкой и начислением баллов

### 3. Фильтрация семейных задач
Механизм фильтрации задач по различным критериям
