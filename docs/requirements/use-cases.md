# Use Cases (Сценарии использования)
## Семейный менеджер задач

### Общая информация
**Система:** Family Task Manager  
**Цель:** Управление семейными задачами, распределение обязанностей и мотивация членов семьи  
**Основные пользователи:** Семейный администратор, Члены семьи

---

## Акторы системы

### 1. Семейный администратор (Family Admin)
**Описание:** Создатель семьи, имеет максимальные права управления  
**Роли:**
- Создание и настройка семьи
- Приглашение/удаление членов семьи
- Управление категориями задач
- Просмотр полной статистики

### 2. Член семьи (Family Member)
**Описание:** Участник семьи, выполняет задачи  
**Роли:**
- Просмотр задач семьи
- Выполнение назначенных задач
- Создание новых задач
- Просмотр личной статистики

### 3. Система (System)
**Описание:** Автоматические процессы системы  
**Роли:**
- Отправка уведомлений и напоминаний
- Начисление баллов за задачи
- Генерация отчетов

---

## Основные сценарии использования

### UC-01: Создание семьи
**Актор:** Семейный администратор  
**Предусловия:** Пользователь зарегистрирован в системе  
**Основной поток:**
1. Пользователь выбирает "Создать новую семью"
2. Вводит название семьи и описание
3. Система создает семью и назначает пользователя администратором
4. Система генерирует пригласительный код для семьи
**Постусловия:** Создана новая семья, пользователь становится администратором  
**Связанные диаграммы:** [Use Case Diagram](../../behavior/use_case/use_case_diagram.png)  
**Связанные требования:** FR-01, FR-02

### UC-02: Приглашение члена семьи
**Актор:** Семейный администратор  
**Предусловия:** Семья создана, администратор авторизован  
**Основной поток:**
1. Администратор выбирает "Пригласить в семью"
2. Вводит email или генерирует пригласительную ссылку
3. Система отправляет приглашение
4. Приглашенный пользователь принимает приглашение
**Альтернативный поток A:** Пользователь уже в семье - показывается ошибка  
**Связанные диаграммы:** [Use Case Diagram](../../behavior/use_case/use_case_diagram.png)  
**Связанные требования:** FR-03

### UC-03: Создание семейной задачи
**Актор:** Член семьи (или администратор)  
**Предусловия:** Пользователь состоит в семье  
**Основной поток:**
1. Пользователь выбирает "Создать задачу"
2. Заполняет форму: название, описание, категория, приоритет, срок, ответственный
3. Система валидирует данные
4. Задача создается в системе
5. Ответственному отправляется уведомление
**Связанные диаграммы:** [Sequence Diagram](../../behavior/sequence/create_family_task.png)  
**Связанные требования:** FR-04, FR-05, NFR-01

### UC-04: Отметка задачи выполненной
**Актор:** Член семьи  
**Предусловия:** Задача назначена на пользователя, не выполнена  
**Основной поток:**
1. Пользователь отмечает задачу как выполненную
2. При необходимости прикрепляет фото подтверждения
3. Система проверяет выполнение
4. Начисляются баллы за выполнение
5. Обновляется статистика пользователя и семьи
**Альтернативный поток A:** Требуется подтверждение администратора  
**Связанные диаграммы:** [Sequence Diagram](../../behavior/sequence/complete_task_verification.png)  
**Связанные требования:** FR-06, FR-07

### UC-05: Фильтрация и поиск задач
**Актор:** Член семьи  
**Предусловия:** Пользователь авторизован и состоит в семье  
**Основной поток:**
1. Пользователь открывает список задач
2. Применяет фильтры: статус, категория, приоритет, ответственный
3. Система показывает отфильтрованный список
4. Пользователь может сортировать результаты
**Связанные диаграммы:** [Sequence Diagram](../../behavior/sequence/filter_family_tasks.png)  
**Связанные требования:** FR-08, NFR-02

### UC-06: Просмотр статистики и отчетов
**Актор:** Член семьи (базовая), Администратор (расширенная)  
**Предусловия:** Пользователь состоит в семье  
**Основной поток:**
1. Пользователь открывает раздел "Статистика"
2. Система показывает:
   - Личные баллы и достижения
   - Выполненные задачи за период
   - Активность семьи
3. Администратор видит дополнительно:
   - Статистику по всем членам семьи
   - Эффективность распределения задач
**Связанные диаграммы:** [Daily Workflow Sequence](../../behavior/sequence/daily_workflow.png) *(если создадите)*  
**Связанные требования:** FR-09

### UC-07: Автоматические уведомления
**Актор:** Система  
**Триггер:** События в системе (создание задачи, приближение дедлайна и т.д.)  
**Основной поток:**
1. Система обнаруживает событие, требующее уведомления
2. Формируется сообщение
3. Отправляется уведомление через выбранный канал (email, push)
4. Логируется факт отправки
**Типы уведомлений:**
- Новая задача назначена
- Напоминание о сроке выполнения
- Задача выполнена (для администратора)
- Достижение достигнуто
**Связанные требования:** NFR-03, NFR-04

### UC-08: Управление категориями задач
**Актор:** Семейный администратор  
**Предусловия:** Администратор авторизован  
**Основной поток:**
1. Администратор открывает управление категориями
2. Создает/редактирует/удаляет категории
3. Настраивает иконки и цвета категорий
4. Система обновляет категории для всех задач
**Ограничения:** Нельзя удалить категорию, если есть задачи в ней  
**Связанные требования:** FR-10

## Специальные сценарии

### UC-09: Делегирование задачи
**Актор:** Член семьи  
**Предусловия:** Задача назначена на пользователя  
**Основной поток:**
1. Пользователь выбирает задачу для делегирования
2. Выбирает нового ответственного из членов семьи
3. Указывает причину делегирования
4. Система переназначает задачу, отправляет уведомления
**Ограничения:** Можно делегировать только свои задачи

### UC-10: Повторяющиеся задачи
**Актор:** Семейный администратор  
**Предусловия:** Администратор авторизован  
**Основной поток:**
1. Администратор создает задачу с типом "Повторяющаяся"
2. Настраивает расписание (ежедневно, еженедельно, ежемесячно)
3. Система автоматически создает экземпляры задач по расписанию
4. При выполнении - создается следующая итерация
**Примеры:** Вынос мусора, Покупка продуктов, Уборка

## Глоссарий

| Термин | Определение |
|--------|-------------|
| Семья | Группа пользователей, объединенных для совместного управления задачами |
| Задача | Единица работы, которая должна быть выполнена |
| Баллы | Внутренняя валюта системы для мотивации |
| Категория | Группа для классификации задач |
| Администратор | Пользователь с правами управления семьей |
| Дедлайн | Крайний срок выполнения задачи |
